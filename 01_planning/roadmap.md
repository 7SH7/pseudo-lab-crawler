## Phase 1: 데이터 수집 (예상 기간: 데이터 양에 따라 가변)

### 1.1 데이터 소스 및 전략

| 소스 | 기간 | 비고 |
|------|------|------|
| **arXiv** | 1991~1999 | arXiv는 1991년 시작, q-bio 카테고리는 2003년 신설 |
| **Semantic Scholar** | 1970~1999 | 주력 소스, API 제공 |

> **중요**: arXiv의 q-bio 카테고리는 2003년에 신설되었으므로, 고전 생명과학(1970~1999)은 **Semantic Scholar가 주력 소스**가 됩니다.

### 1.2 수집 파이프라인

```
[1] 메타데이터 수집 (title, abstract, authors, year, doi 등)
         ↓
[2] 전문(Full-text) 수집 시도 (가능한 경우만)
         ↓
[3] 중복 제거 (title + abstract 기반)
         ↓
[4] .parquet 형식으로 저장
```

### 1.3 예상 데이터 규모

- Semantic Scholar에서 1970~1999 생명과학 논문: **수십만~수백만 건**
- 실제 full-text 확보 가능 비율: **10~30%** (나머지는 abstract only)

## Phase 2: 데이터 전처리

### 2.1 텍스트 클리닝
- LaTeX 수식 보존 (있는 경우)
- HTML 태그 제거
- 특수문자 정규화

### 2.2 품질 필터링
- 너무 짧은 abstract 제외 (< 100 words)
- 영어 외 언어 필터링 (선택적)
- medicine/healthcare 관련 키워드 제외

### 2.3 중복 제거
- 1차: Title 유사도 >= 0.9
- 2차: Abstract 유사도 >= 0.85

---

## Phase 3: CPT (Continual Pre-Training) 중형 모델

### 3.1 데이터 믹싱 전략

| 데이터 타입 | 비율 | 설명 |
|------------|------|------|
| 과학 도메인 데이터 | 80~85% | 생명과학 논문 + 다른 팀원 데이터 |
| 일반 도메인 데이터 | 15~20% | 웹 텍스트, 코드 등 |

### 3.2 학습 목표
- Next-token prediction
- 과학 도메인 지식 주입
- Catastrophic forgetting 방지

### 3.3 평가 지표
- Perplexity on held-out science corpus
- 과학 QA 벤치마크 성능
